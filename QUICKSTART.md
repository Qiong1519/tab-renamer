# 快速开始指南

## 🚀 5分钟快速上手

### 第一步：生成图标（可选）

扩展需要PNG图标才能正确显示。如果暂时跳过此步骤，Chrome会使用默认图标。

#### 方法 1：使用 Node.js 脚本（推荐）

```bash
# 安装依赖
npm install sharp

# 生成图标
npm run generate-icons
```

#### 方法 2：使用在线工具

1. 访问 https://cloudconvert.com/svg-to-png
2. 上传 `icons/icon.svg`
3. 分别导出为 16x16、48x48、128x128 像素的 PNG
4. 下载并重命名为 `icon16.png`, `icon48.png`, `icon128.png`
5. 保存到 `icons/` 目录

#### 方法 3：临时跳过

如果暂时不想生成图标，可以直接加载扩展，Chrome会使用默认图标。

### 第二步：加载扩展

1. 打开 Chrome 浏览器
2. 在地址栏输入 `chrome://extensions/` 并回车
3. 开启右上角的 **"开发者模式"** 开关
4. 点击 **"加载已解压的扩展程序"** 按钮
5. 选择 `TabRenamer` 项目根目录（包含 manifest.json 的目录）
6. 完成！扩展图标会出现在浏览器工具栏中

### 第三步：使用扩展

#### 快速修改标题

1. 打开任意网页（例如 https://github.com）
2. 点击浏览器工具栏中的 TabRenamer 图标
3. 在弹出的窗口中：
   - 选择修改方式（替换/前缀/后缀）
   - 输入内容（例如："[GitHub] "）
   - 点击"应用修改"
4. 标签页标题立即更新！

#### 创建自动规则

1. 在 Popup 窗口中点击"保存为规则"
2. 在打开的规则管理页面中：
   - 输入规则名称（例如："GitHub项目"）
   - 查看匹配条件（默认使用当前页面的域名）
   - **点击"测试规则"查看效果**
   - 调整设置，满意后点击"保存规则"
3. 规则创建成功！
4. 刷新页面或访问其他 GitHub 页面，标题会自动修改

## 📝 使用示例

### 示例 1：标记工作相关网站

```
规则名称：工作网站
匹配目标：URL
匹配方式：正则表达式
匹配模式：.*\.company\.com
修改动作：添加前缀
修改内容：💼 
```

效果：所有公司网站的标题前会添加工作表情

### 示例 2：简化过长的标题

```
规则名称：简化标题
匹配目标：标题
匹配方式：正则表达式
匹配模式：^(.{30}).*$
修改动作：正则替换
修改内容：$1...
```

效果：标题超过30字符时会被截断并添加省略号

### 示例 3：添加时间戳

```
规则名称：带时间戳
匹配目标：URL
匹配方式：包含
匹配模式：docs.google.com
修改动作：添加前缀
修改内容：[$time] 
```

效果：Google Docs 页面标题会添加当前时间

**注意**：MVP版本暂不支持 `$time` 和 `$date` 变量，后续版本会添加

## 🔧 常见问题

### Q: 扩展无法加载？

检查以下几点：
1. 确保选择的是项目根目录（包含 manifest.json）
2. 确保开启了"开发者模式"
3. 查看扩展页面的错误提示

### Q: 修改标题后刷新页面又恢复了？

这是正常的：
- 如果页面匹配了某条规则，刷新后会自动应用规则
- 如果没有匹配的规则，标题会恢复为原始内容
- 要永久保持修改，请创建规则

### Q: 某些网站无法修改标题？

Chrome 限制了以下页面的修改：
- `chrome://` 开头的内部页面
- `chrome-extension://` 扩展页面
- Chrome Web Store
- 其他安全限制页面

遇到这些页面时，扩展会显示"此页面不支持修改"的提示。

### Q: 规则创建后不生效？

检查以下几点：
1. 规则是否已启用（开关为蓝色）
2. 匹配条件是否正确（使用"测试规则"功能验证）
3. 尝试刷新页面
4. 检查是否有多条规则冲突（只有第一条匹配的规则会生效）

## 💡 小贴士

1. **使用测试功能**：创建规则前总是先测试，确保效果符合预期
2. **命名清晰**：给规则起一个描述性的名字，方便后续管理
3. **合理使用前缀**：添加简洁的前缀或后缀，不要让标题变得更长
4. **正则表达式**：掌握基础正则可以让匹配更精准
5. **导出备份**：定期导出规则数据，避免丢失
